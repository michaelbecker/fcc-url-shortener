FreeCodeCamp URL Shortener Project

User Stories

  • I can pass a URL as a parameter and I will receive a shortened URL in the JSON response.
  • If I pass an invalid URL that doesn't follow the valid http://www.example.com format, the JSON response will contain an error instead.
  • When I visit that shortened URL, it will redirect me to my original link.

Examples

Good URL

https://almond-editor.glitch.me/new/http://www.google.com
{"original_url":"http://www.google.com","short_url":"https://almond-editor.glitch.me/r/2702955352"}

Bad URL

https://almond-editor.glitch.me/new/invalid_url
{"error":"Invalid URL format"}

Usage

Enter the following as a web address:
https://almond-editor.glitch.me/r/2702955352"

And you will be redirected to
http://www.google.com.

Discussion

The hashing algorithm for this demo project is simple, we hash the URL string and use that as the shorter web address. There are several areas for improvement if this was a real URL shortening application. First, the hash library used only outputs numbers, which makes the "short" URL longer than it needs be. We could investigate a hash that outputs alphanumeric values instead, or converts the hash into chars where possible which would also shorten the hash. Next, no consideration has been made as to whether there are collisions with a hash for two unique URLs. While it's unlikely, it's still nevertheless possible. Therefore in a production application, we should be checking for hash collisions and expanding the "short" URL to handle this.
